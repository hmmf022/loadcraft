# 開発ロードマップ

本ドキュメントでは、コンテナ積載シミュレータの開発を 5 つのフェーズに分割し、各フェーズの実装ステップ・成果物・検証方法を定義する。

プロトタイプファーストのアプローチを採用し、Phase 1 で動作する最小限のアプリケーションを構築したうえで、段階的に機能を拡充する。

技術スタック: **TypeScript + React + Vite + WebGPU**

---

## 1. フェーズ概要

| Phase | 名称 | 概要 | 主要成果物 |
|-------|------|------|-----------|
| **Phase 1** | プロトタイプ（基盤構築） | Core データ構造、基本レンダリング、基本 UI | 動作するプロトタイプ（手動配置） |
| **Phase 2** | インタラクション | ドラッグ＆ドロップ、選択、レイキャスト | 完全なドラッグ＆ドロップワークフロー |
| **Phase 3** | 任意回転・高度な機能 | 任意角度回転、カメラプリセット、カスタムコンテナ | 回転対応・カメラ拡張 |
| **Phase 4** | データ分析・永続化 | 重量・重心計算、統計、セーブ/ロード | 重量バランス分析・ファイル入出力 |
| **Phase 5** | ビジュアル・UX 改善 | ラベル、ゴースト描画、アニメーション、レスポンシブ | 完成度の高いビジュアルと UX |

```
Phase 1          Phase 2          Phase 3          Phase 4          Phase 5
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ Core     │    │ Drag &   │    │ Rotation │    │ Weight & │    │ Labels & │
│ Rendering│───>│ Drop     │───>│ Camera   │───>│ Analysis │───>│ Polish   │
│ Basic UI │    │ Select   │    │ Custom   │    │ Save/Load│    │ Responsive│
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
  基盤構築        操作性確立       高度な機能       データ活用       仕上げ
```

各フェーズは前のフェーズの成果物に依存するため、原則として順番に実施する。ただし、Phase 内のステップは並行して進めてもよい場合がある。

## 2. Phase 1: プロトタイプ（基盤構築）

Phase 1 の目標は、コンテナ内に貨物を手動配置できる最小限のプロトタイプを構築することである。Core データ構造、WebGPU レンダリングパイプライン、基本 UI の 3 要素を段階的に組み上げる。

### 2.1 ステップ詳細

#### ステップ 1: プロジェクト初期化

Vite + React + TypeScript のプロジェクトを作成し、開発環境を整備する。

- Vite + React + TypeScript のプロジェクトセットアップ
- WebGPU 型定義（`@webgpu/types`）のインストール
- ディレクトリ構造の作成（`src/core/`、`src/renderer/`、`src/ui/`、`src/state/`）
- ESLint / Prettier の設定
- **成果物**: `npm run dev` で開発サーバーが起動する

```
container-simulator/
├── src/
│   ├── core/           # ドメインロジック
│   ├── renderer/       # WebGPU 描画
│   ├── ui/             # React コンポーネント
│   ├── state/          # Zustand ストア
│   └── main.tsx
├── public/
├── tests/
├── package.json
├── tsconfig.json
└── vite.config.ts
```

#### ステップ 2: Core データ構造

型定義とボクセルグリッドを実装する。

- `types.ts` の実装（全インターフェース: `ContainerDef`、`CargoItemDef`、`PlacedCargo`、`Vec3` 等）
- `VoxelGrid` クラスの実装
  - コンストラクタ（コンテナサイズからグリッド生成）
  - `get(x, y, z)`: 指定座標のボクセル値を取得
  - `set(x, y, z, value)`: 指定座標にボクセル値を設定
  - `fillBox(position, size, objectId)`: 直方体領域を objectId で埋める
  - `clearObject(objectId)`: 指定 objectId のボクセルをすべてクリア
- **成果物**: VoxelGrid のユニットテストがすべてパスする

#### ステップ 3: WebGPU 初期化

WebGPU レンダリングパイプラインの基盤を構築する。

- `Renderer.ts` の実装（初期化シーケンス）
  - Canvas 要素のセットアップ
  - GPU アダプター・デバイスの取得
  - レンダーパイプラインの作成（頂点シェーダー + フラグメントシェーダー）
  - 深度バッファの設定
- 単一の色付きキューブをレンダリングしてパイプラインの動作を検証する
- **成果物**: 画面上に色付きキューブが表示される

#### ステップ 4: カメラシステム

オービットカメラを実装し、3D シーンを自由に観察できるようにする。

- `OrbitCamera` の実装
  - 球面座標系（`theta`、`phi`、`radius`）
  - ビュー行列（`lookAt`）の計算
  - 射影行列（`perspective`）の計算
- `CameraController` の実装
  - マウスドラッグでオービット（左ボタン）
  - マウスホイールでズーム
  - 中ボタンドラッグでパン
- **成果物**: マウス操作でキューブの周りを自由にオービットできる

#### ステップ 5: インスタンスド貨物レンダリング

複数の貨物を効率的にレンダリングするインスタンスドレンダリングを実装する。

- `CargoPipeline` の実装
  - インスタンスストレージバッファの作成
  - インスタンスごとの位置・サイズ・色の管理
  - 頂点シェーダーでのインスタンスデータの読み取り
- 異なる位置・色で複数のボックスをレンダリングする
- **成果物**: 複数の色付きボックスが効率的にレンダリングされる

#### ステップ 6: コンテナ壁レンダリング

コンテナの壁面を半透明で描画する特殊パイプラインを実装する。

- `ContainerPipeline` の実装
  - 透過処理（アルファブレンディング）
  - 2 つのサブパス: 内側壁面レンダリング + 外側壁面レンダリング
  - 内側から見ると不透明、外側から見ると半透明の切り替え
- **成果物**: コンテナ壁が表示され、外側から見たときに内部が透けて見える

#### ステップ 7: フロアグリッド

コンテナ床面にグリッド線を描画する。

- `GridPipeline` の実装
  - コンテナ床面のサイズに合わせたグリッド生成
  - グリッド線の色・太さの設定
- **成果物**: コンテナ床面にグリッドが表示される

#### ステップ 8: 基本 UI

React コンポーネントによるサイドバー UI を実装する。

- `App` レイアウト（サイドバー + キャンバスの分割レイアウト）
- `ContainerSelector`（プリセット選択のみ、カスタムサイズは Phase 3）
  - 20ft / 40ft コンテナのプリセット
- `CargoEditor`（基本フォーム）
  - 名称、幅・高さ・奥行き、色、重量の入力
- `CargoList`（表示のみ、ドラッグは Phase 2）
  - 登録済み貨物定義の一覧表示
- **成果物**: サイドバーで貨物定義を作成できる

#### ステップ 9: 統合 -- 手動配置

UI と Core を接続し、ボタン操作で貨物をコンテナ内に配置できるようにする。

- 「配置」ボタンのクリックで貨物を `(0, 0, 0)` に追加する
- VoxelGrid の更新処理
- レンダラーの再描画トリガー（`renderVersion` のインクリメント）
- 衝突判定の基本実装（同じ座標への重複配置を検出）
- **成果物**: サイドバーのボタンからコンテナに貨物を追加でき、3D ビューに表示される

### 2.2 成果物

- WebGPU で 3D レンダリングされたコンテナと貨物が表示される動作可能なプロトタイプ
- マウス操作によるオービットカメラ
- サイドバーからの貨物定義作成と手動配置
- 以降のすべてのフェーズの基盤となるコード構造

## 3. Phase 2: インタラクション

Phase 2 の目標は、ドラッグ＆ドロップによる直感的な貨物配置と、オブジェクトの選択・操作・履歴管理を実現することである。

### 3.1 ステップ詳細

#### ステップ 1: レイキャスターの実装

マウス座標から 3D シーンへのレイを計算する。

- スクリーン座標からワールド空間へのレイ変換
- レイと AABB（軸平行バウンディングボックス）の交差判定
- 最も手前のオブジェクトの特定
- レイとコンテナ床面の交差座標の計算

#### ステップ 2: オブジェクト選択

クリックによるオブジェクト選択機能を実装する。

- クリック時にレイキャストを実行し、ヒットした貨物を選択状態にする
- 選択された貨物のハイライト表示（色の変更またはアウトライン）
- 空白領域のクリックで選択解除
- ストアの `selectedObjectId` を更新する

#### ステップ 3: ドラッグ＆ドロップ（CargoList → キャンバス）

CargoList からキャンバスへの貨物配置をドラッグ＆ドロップで行えるようにする。

- HTML5 Drag API の `dragstart` で `cargoDefId` を転送
- キャンバス上の `dragover` でドロップを許可
- `drop` イベントでレイキャストにより配置座標を決定
- VoxelGrid の衝突チェックを行い、配置可否を判定する

#### ステップ 4: ゴーストレンダリング（ドラッグ中のプレビュー）

ドラッグ中に配置先のプレビューを半透明で表示する。

- ドラッグ中のマウス位置に応じてゴーストオブジェクトを描画
- 衝突がない場合は緑色半透明、衝突がある場合は赤色半透明で表示
- ドラッグ終了時にゴーストを消去する

#### ステップ 5: 3D 空間でのオブジェクト移動

配置済みの貨物を 3D 空間内で移動できるようにする。

- 選択中の貨物をドラッグして移動
- 移動中は元の位置のボクセルをクリアし、ゴースト表示する
- ドロップ時に新しい位置で VoxelGrid を更新
- 重力適用（浮遊禁止）: Y 軸方向に最も低い有効位置に自動スナップ

#### ステップ 6: 選択オブジェクトの削除

選択中のオブジェクトを削除する機能を実装する。

- Delete / Backspace キーで選択中の貨物を削除
- VoxelGrid から該当オブジェクトのボクセルをクリア
- `PlacedCargo[]` から該当エントリを削除
- レンダラーの再描画をトリガー

#### ステップ 7: Undo / Redo（History モジュール）

操作の取り消し・やり直しを実装する。

- `History` モジュールの実装
  - 状態スナップショットのスタック管理
  - Undo スタックと Redo スタックの分離
  - Undo 上限の設定（デフォルト: 50 回）
  - 新しいアクション実行時に Redo スタックをクリア
- Ctrl+Z / Ctrl+Y（Ctrl+Shift+Z）のキーバインディング

#### ステップ 8: キーボードショートカット

主要な操作にキーボードショートカットを割り当てる。

- `Delete` / `Backspace`: 選択オブジェクト削除
- `Ctrl+Z`: Undo
- `Ctrl+Y` / `Ctrl+Shift+Z`: Redo
- `Escape`: 選択解除 / ドラッグキャンセル
- `Ctrl+A`: 全選択（将来的な複数選択への拡張に備える）

### 3.2 成果物

- CargoList からキャンバスへのドラッグ＆ドロップによる貨物配置
- クリックによるオブジェクト選択とハイライト表示
- 配置済みオブジェクトの 3D 空間内移動
- オブジェクト削除
- Undo / Redo による操作履歴管理
- キーボードショートカット

## 4. Phase 3: 任意回転・高度な機能

Phase 3 の目標は、貨物の任意角度回転、カメラプリセット、カスタムコンテナサイズ、グリッドスナッピングを実装することである。

### 4.1 ステップ詳細

#### ステップ 1: Voxelizer モジュール

任意角度で回転した貨物をボクセル化するモジュールを実装する。

- 回転行列の生成（オイラー角 → 回転行列）
- 回転後の AABB（軸平行バウンディングボックス）の計算
- OBB（有向バウンディングボックス）テストによるボクセル充填
  - 回転後の各ボクセル位置が元の直方体内に含まれるかを判定
- ボクセル化結果のキャッシュ機構

#### ステップ 2: 回転 UI コントロール

PlacementControls に回転操作用の UI を追加する。

- X / Y / Z 軸ごとの回転角度入力（数値入力 or スライダー）
- 90° 単位のクイック回転ボタン
- 回転プレビューの即時反映

#### ステップ 3: キーボード回転ショートカット

キーボードで素早く回転操作を行えるようにする。

- `R`: Y 軸周りに 90° 回転
- `Shift+R`: Y 軸周りに -90° 回転
- `T`: X 軸周りに 90° 回転
- `F`: Z 軸周りに 90° 回転
- 任意角度の回転は UI からのみ操作可能とする

#### ステップ 4: ドラッグ＆ドロップ中の回転

ドラッグ中にも回転操作を適用できるようにする。

- ドラッグ中にキーボードショートカットで回転
- ゴーストレンダリングに回転を反映
- 回転後の衝突判定を即時再計算

#### ステップ 5: 固定カメラビュー

プリセットされたカメラアングルへのワンクリック切り替えを実装する。

- 正面（Front）
- 背面（Back）
- 左側面（Left）
- 右側面（Right）
- 上面（Top）
- アイソメトリック（Isometric）
- ツールバーまたはキーバインドで切り替え

#### ステップ 6: カスタムコンテナサイズ入力

ユーザーが任意のコンテナサイズを指定できるようにする。

- `ContainerSelector` に「カスタム」オプションを追加
- 幅・高さ・奥行きの数値入力フォーム
- 入力値のバリデーション（最小・最大サイズ制限）
- サイズ変更時の VoxelGrid 再生成

#### ステップ 7: グリッドスナッピング

貨物配置時にグリッドにスナップする機能を実装する。

- スナップ間隔の設定（1cm / 5cm / 10cm）
- ドラッグ中の配置座標をスナップ間隔に丸める
- スナップの ON/OFF 切り替え

### 4.2 成果物

- 任意角度での貨物回転とボクセル化
- 回転 UI コントロールとキーボードショートカット
- ドラッグ＆ドロップ中の回転操作
- 6 種類の固定カメラビュー
- カスタムコンテナサイズ入力
- グリッドスナッピング

## 5. Phase 4: データ分析・永続化

Phase 4 の目標は、重量・重心の分析機能、警告表示、セーブ/ロード、インポート機能を実装することである。

### 5.1 ステップ詳細

#### ステップ 1: WeightCalculator モジュール

コンテナ内の貨物の総重量と重心を計算する。

- 個別貨物の重量計算
- 全貨物の総重量計算
- 重心位置（Center of Gravity）の算出
  - 各貨物の重心（配置位置 + サイズの中心）× 重量の加重平均

#### ステップ 2: GravityChecker モジュール

貨物が物理的に安定しているかを判定する。

- 床面支持の判定: 貨物が床面（Y=0）に接しているか
- 部分支持の判定: 貨物の底面がほかの貨物の上面で支えられているか
- 浮遊検出: 底面にまったく支持がない貨物を検出
- 支持率の計算: 底面のうち何パーセントが支持されているか

#### ステップ 3: StatsPanel

統計情報を表示するパネルを実装する。

- 総重量 / 最大積載重量
- 重心位置（X, Y, Z）の表示
- 充填率（使用体積 / コンテナ体積 × 100%）
- 貨物数の表示

#### ステップ 4: フローティング警告表示

3D ビュー上に警告メッセージをオーバーレイ表示する。

- 浮遊している貨物の警告（該当貨物をハイライト）
- 重心位置が偏っている場合の警告
- 衝突（重なり）が発生している箇所の警告

#### ステップ 5: 過積載警告

コンテナの最大積載重量を超過した場合の警告を実装する。

- `ContainerDef` に `maxPayload`（最大積載重量）を追加
- 総重量が `maxPayload` を超えた場合に警告を表示
- StatsPanel の重量表示を赤色に変更

#### ステップ 6: JSON セーブ/ロード

現在の配置状態を JSON ファイルとして保存・読み込みする。

- セーブデータのフォーマット定義（バージョン番号を含む）
- `File` API によるダウンロード（セーブ）
- `FileReader` API による読み込み（ロード）
- ロード時のバリデーション（フォーマット検証、バージョンチェック）
- localStorage への自動保存（データ消失防止）

#### ステップ 7: CSV / JSON インポート

貨物定義を CSV または JSON ファイルからインポートする。

- CSV パーサーの実装（ヘッダー行の検出、列マッピング）
- JSON 配列形式のインポート
- バリデーション（必須列の確認、型チェック、値の範囲チェック）
- インポート結果のプレビュー表示
- 文字エンコーディング対応（UTF-8 / Shift_JIS）

### 5.2 成果物

- 重量・重心の自動計算と表示
- 浮遊・過積載・重心偏りの警告表示
- JSON 形式でのセーブ/ロード
- CSV / JSON からの貨物定義インポート
- localStorage による自動保存

## 6. Phase 5: ビジュアル・UX 改善

Phase 5 の目標は、視覚的な完成度と UX を向上させ、パフォーマンスを最適化することである。

### 6.1 ステップ詳細

#### ステップ 1: LabelRenderer（3D テキストラベル）

貨物上に名称や寸法を表示するラベル描画機能を実装する。

- 各貨物の上面にテキストラベルを描画
- 表示内容: 貨物名、寸法（W×H×D）、重量
- カメラ距離に応じたラベルの表示/非表示切り替え
- ラベル同士の重なり回避

#### ステップ 2: スムーズなカメラ遷移

カメラビュー切り替え時にスムーズなアニメーションを適用する。

- 固定ビュー間の遷移にイージングを適用
- 球面線形補間（Slerp）によるスムーズな回転
- アニメーション時間の設定（デフォルト: 300ms）

#### ステップ 3: ゴーストレンダリングの改善

ドラッグ中のプレビュー表示をより直感的にする。

- 透明度のグラデーション（距離に応じて変化）
- 配置可否のカラーコーディング
  - 緑: 配置可能（衝突なし）
  - 赤: 配置不可（衝突あり）
  - 黄: 配置可能だが浮遊状態
- ゴーストのエッジラインの描画

#### ステップ 4: 選択アウトライン・ハイライト効果

選択されたオブジェクトの視覚的強調をより洗練されたものにする。

- アウトラインポストエフェクトの実装
- 選択時のパルスアニメーション（微細な拡大縮小）
- ホバー時のハイライト（マウスオーバーでの軽い強調）

#### ステップ 5: レスポンシブレイアウト

さまざまな画面サイズに対応するレスポンシブデザインを実装する。

- サイドバーの折りたたみ/展開
- キャンバスサイズの動的調整
- ブレークポイントに応じたレイアウト切り替え
- タッチデバイスへの基本対応（将来の拡張に備える）

#### ステップ 6: ローディング状態とエラーメッセージ

ユーザーへのフィードバックを充実させる。

- WebGPU 初期化中のローディング表示

- ファイル読み込み中のプログレスバー
- 操作エラー時のトースト通知

#### ステップ 7: パフォーマンス最適化

大規模なコンテナや多数の貨物でのパフォーマンスを改善する。

- 大規模コンテナ（40ft: 約 1,200 × 240 × 240 cm）への対応

- Web Workers によるボクセル演算のオフロード
- メモリ使用量のモニタリングと警告

## 7. 検証方法

### 7.1 ユニットテスト対象

各モジュールについて、以下のテスト項目を実装する。

| モジュール | テスト項目 |
|-----------|-----------|
| **VoxelGrid** | `get` / `set` の正常動作、`fillBox` による領域充填、`clearObject` による全クリア、`hasCollision` による衝突判定、`isSupported` による支持判定、境界値チェック（範囲外アクセス） |
| **Voxelizer** | 軸平行状態でのボクセル化、45° 回転でのボクセル化、任意角度回転でのボクセル化、回転後 AABB の正確な計算 |
| **GravityChecker** | 床面支持の判定、部分支持の判定、浮遊状態の検出、支持率の計算精度 |
| **WeightCalculator** | 単一貨物の重量計算、複数貨物の総重量計算、重心位置の計算精度 |
| **History** | Undo 実行と状態復元、Redo 実行と状態復元、Undo 上限（50 回）の動作、新規アクション実行時の Redo スタッククリア |
| **CSV パーサー** | 正常な CSV の解析、必須列の欠落検出、不正な値の検出、文字エンコーディング（UTF-8 / Shift_JIS）の対応 |
| **ファイル I/O** | セーブフォーマットの正確性、ロード時のバリデーション、バージョンチェック（旧バージョンの互換性） |

### 7.2 手動テスト項目

自動テストでカバーしにくい以下の項目は、手動テストで検証する。

- **WebGPU 初期化**: Chrome および Edge での正常な初期化を確認
- **カメラ操作**: オービット、パン、ズームのスムーズさを確認
- **ドラッグ＆ドロップ**: 配置座標の正確性、ゴーストのレスポンスを確認
- **コンテナ壁透過**: さまざまなカメラアングルでの透過表示の正確性を確認
- **Undo / Redo**: 複数操作（配置・移動・削除・回転）の組み合わせで正しく動作するか確認
- **セーブ → ロード**: 保存後に再読み込みし、同一の状態が復元されることを確認
- **大規模コンテナ**: 40ft コンテナ（約 1,200 × 240 × 240 cm）での動作を確認
- **多数貨物レンダリング**: 50 個以上の貨物を配置した場合の描画パフォーマンスを確認

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| **WebGPU ブラウザサポート** | 非対応ブラウザのユーザーがアプリを利用できない | 明確なエラーメッセージを表示し、対応ブラウザ（Chrome 113+、Edge 113+）を案内する。WebGPU の利用可否を起動時にチェックする |
| **メモリ使用量（40ft コンテナ ≈ 145MB）** | ブラウザタブのクラッシュ | メモリ使用量を監視し、大規模コンテナの選択時に警告を表示する。必要に応じて 5cm 解像度へのフォールバックを検討する |
| **Voxelizer パフォーマンス（任意回転）** | 配置操作の遅延 | ボクセル化結果をキャッシュする。大規模オブジェクトの場合は Web Workers でバックグラウンド処理を行う |
| **レンダリングパフォーマンス（多数オブジェクト）** | FPS 低下によるインタラクション品質の劣化 | インスタンスドレンダリングの活用、フラスタムカリング、LOD の段階的導入 |
| **ドラッグ＆ドロップの UX** | ユーザー体験の低品質化 | 反復的な UX テストと改善、スムーズなゴーストレンダリング、明確なフィードバック表示 |
| **データ消失** | ユーザーの作業内容が失われる | localStorage への自動保存（一定間隔）、明示的な保存リマインダーの表示、ブラウザ閉じる前の確認ダイアログ |
