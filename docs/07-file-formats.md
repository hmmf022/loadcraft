# 07 - ファイル形式仕様

本ドキュメントでは、コンテナ積載シミュレータで使用するファイル形式を定義する。保存・読み込み用のJSON形式、および荷物定義インポート用のCSV・JSON形式を対象とする。

> **単位規約**: 寸法はすべてセンチメートル (cm)、重量はキログラム (kg)、回転はオイラー角の度数 (deg) で表現する。

---

## 1. 保存JSON形式（PlacementState）

配置状態の永続化に使用するJSON形式のスキーマを以下に示す。

```json
{
  "version": "1.0",
  "savedAt": "2026-02-18T12:00:00Z",
  "container": {
    "widthCm": 590,
    "heightCm": 239,
    "depthCm": 235,
    "maxPayloadKg": 28200
  },
  "cargoDefs": [
    {
      "id": "uuid-1234",
      "name": "段ボールA",
      "widthCm": 60,
      "heightCm": 50,
      "depthCm": 40,
      "weightKg": 25.5,
      "color": "#FF6B35"
    }
  ],
  "placements": [
    {
      "instanceId": 1,
      "cargoDefId": "uuid-1234",
      "positionCm": { "x": 0, "y": 0, "z": 0 },
      "rotationDeg": { "x": 0, "y": 0, "z": 0 }
    },
    {
      "instanceId": 2,
      "cargoDefId": "uuid-1234",
      "positionCm": { "x": 60, "y": 0, "z": 0 },
      "rotationDeg": { "x": 0, "y": 45, "z": 0 }
    }
  ],
  "nextInstanceId": 3
}
```

### 1.1 スキーマ詳細

| フィールド | 型 | 説明 |
|-----------|------|------|
| `version` | string | 形式バージョン。将来の互換性確保のために使用する。現在は `"1.0"` 固定。 |
| `savedAt` | string | ISO 8601 形式の保存日時（例: `"2026-02-18T12:00:00Z"`）。 |
| `container` | ContainerDef | コンテナの定義オブジェクト。`widthCm`、`heightCm`、`depthCm`、`maxPayloadKg` を含む。 |
| `cargoDefs` | CargoItemDef[] | 定義済みのすべての荷物タイプの配列。各要素は `id`、`name`、`widthCm`、`heightCm`、`depthCm`、`weightKg`、`color` を持つ。 |
| `placements` | Placement[] | 配置済み荷物の配列。ボクセルデータは含まない（読み込み時に再計算される）。 |
| `nextInstanceId` | number | 次に配置する荷物に割り当てるインスタンスIDのカウンター。 |

**Placement オブジェクト:**

| フィールド | 型 | 説明 |
|-----------|------|------|
| `instanceId` | number | 配置インスタンスの一意識別子。 |
| `cargoDefId` | string | 参照する荷物定義の ID。`cargoDefs` 内の `id` と一致する必要がある。 |
| `positionCm` | `{ x, y, z }` | 配置位置（cm 単位）。コンテナ原点からのオフセット。 |
| `rotationDeg` | `{ x, y, z }` | 回転角度（オイラー角、度数単位）。 |

### 1.2 保存フロー

保存処理は以下の手順で実行する。

1. **シリアライズ**: ストアの状態を JSON に変換する。VoxelGrid およびボクセルキャッシュは除外する（これらは読み込み時に再計算されるため）。
2. **Blob 生成**: JSON 文字列から `Blob` オブジェクトを作成する（`Content-Type: application/json`）。
3. **ダウンロードトリガー**: ブラウザのダウンロードを実行する。ファイル名パターンは `container-sim-YYYY-MM-DD-HHmm.json`（例: `container-sim-2026-02-18-1430.json`）。

```
保存時のファイル名例:
  container-sim-2026-02-18-1430.json
```

### 1.3 読み込みフロー

読み込み処理は以下の手順で実行する。

1. **ファイル選択**: ユーザーがファイルピッカーから JSON ファイルを選択する。
2. **JSON パース**: ファイル内容を JSON としてパースする。
3. **バリデーション**: スキーマバージョンおよび必須フィールドを検証する（詳細は「4. バリデーションルール」を参照）。
4. **コンテナ設定**: `container` の値を設定し、VoxelGrid を再生成する。
5. **荷物定義読み込み**: `cargoDefs` をストアに読み込む。
6. **配置復元**: 各 `placement` に対してボクセライザーでボクセルを再計算し、VoxelGrid に充填する。
7. **カウンター設定**: `nextInstanceId` を設定する。
8. **履歴クリア**: Undo/Redo 履歴をクリアする。

## 2. CSV インポート形式（荷物定義）

荷物定義を一括インポートするための CSV 形式を定義する。

### 2.1 カラム定義

| カラム名 | 型 | 必須 | 説明 |
|---------|------|------|------|
| `name` | string | Yes | 荷物名称 |
| `width_cm` | number | Yes | 幅（cm） |
| `height_cm` | number | Yes | 高さ（cm） |
| `depth_cm` | number | Yes | 奥行き（cm） |
| `weight_kg` | number | Yes | 重量（kg） |
| `color` | string | No | 16進カラーコード（省略時はランダム色を自動付与） |

### 2.2 サンプルCSV

```csv
name,width_cm,height_cm,depth_cm,weight_kg,color
段ボールA,60,50,40,25.5,#FF6B35
段ボールB,80,60,50,35.0,#4ECDC4
パレット,120,15,100,25.0,#C8B560
大型機器,150,180,120,350.0,#E63946
```

### 2.3 パース仕様

| 項目 | 仕様 |
|------|------|
| エンコーディング | UTF-8（BOM あり・なし両対応） |
| デリミタ | カンマ (`,`) |
| ヘッダー行 | 必須（1行目） |
| クォートフィールド | 対応（名前にカンマを含む場合など） |
| パースライブラリ | papaparse（堅牢な CSV パース処理のため） |
| 空行 | スキップする |

**パース処理の流れ:**

1. ファイルを UTF-8 テキストとして読み込む（BOM があれば除去）。
2. papaparse でパースし、ヘッダー行をキーとしてオブジェクト配列に変換する。
3. 各行に対してバリデーションを実行する（詳細は「4. バリデーションルール」を参照）。
4. バリデーションを通過した行について UUID を自動生成し、`CargoItemDef` としてストアに追加する。

## 3. JSON インポート形式（荷物定義）

荷物定義を JSON 形式で一括インポートする場合の形式を定義する。`CargoItemDef` オブジェクトの配列として構成する。

```json
[
  {
    "name": "段ボールA",
    "widthCm": 60,
    "heightCm": 50,
    "depthCm": 40,
    "weightKg": 25.5,
    "color": "#FF6B35"
  },
  {
    "name": "段ボールB",
    "widthCm": 80,
    "heightCm": 60,
    "depthCm": 50,
    "weightKg": 35.0,
    "color": "#4ECDC4"
  }
]
```

**インポート時の注意事項:**

- `id` はインポート時に UUID として自動生成される。JSON 内に `id` フィールドがあっても無視される。
- `color` を省略した場合、ランダムかつ他の荷物と区別しやすい色が自動的に割り当てられる。
- 保存JSON形式（セクション1）とはフィールド構成が異なる点に注意すること。インポート用 JSON は `id` を持たず、配列のルート要素として定義される。

## 4. バリデーションルール

### 4.1 共通ルール

すべてのインポート形式・保存形式に共通するバリデーションルールを以下に示す。

| 対象 | ルール |
|------|--------|
| 寸法値（width, height, depth） | 正の整数であること（`> 0`） |
| 重量（weight） | 非負の数値であること（`>= 0`） |
| カラー（color） | 16進形式に一致すること: `/^#[0-9A-Fa-f]{6}$/` |
| 名前（name） | 空でない文字列であること |

### 4.2 保存JSONバリデーション

保存 JSON ファイルの読み込み時に適用される追加バリデーションルールを以下に示す。

| 対象 | ルール |
|------|--------|
| `version` | `"1.0"` と一致すること |
| `container` | `widthCm`、`heightCm`、`depthCm`、`maxPayloadKg` のすべてのフィールドが存在すること |
| `placements[].cargoDefId` | `cargoDefs` 内に対応する `id` を持つ荷物定義が存在すること |
| `placements[].instanceId` | 配列全体で一意であること |
| `nextInstanceId` | すべての既存 `instanceId` より大きい値であること |

### 4.3 CSVバリデーション

CSV インポート時に適用される追加バリデーションルールを以下に示す。

| 対象 | ルール |
|------|--------|
| ヘッダー行 | 必須カラム（`name`, `width_cm`, `height_cm`, `depth_cm`, `weight_kg`）がすべて含まれていること |
| 数値カラム | 有効な数値としてパースできること |
| エラー報告 | バリデーションエラーが発生した場合、該当する行番号を含めて報告すること |

## 5. エラーハンドリング

各種エラー発生時の処理方針を以下に定義する。

| エラー種別 | 処理方針 |
|-----------|---------|
| **不正な JSON** | パースエラーメッセージとエラー位置を表示する。 |
| **必須フィールド欠損** | 不足しているフィールド名を一覧表示する。 |
| **不正な値** | フィールド名、行番号（CSV の場合）、および期待される形式を表示する。 |
| **部分インポート** | CSV の場合: エラー発生時に、バリデーションを通過した行のみインポートするか選択肢を提示する。保存 JSON の場合: ファイル全体を拒否する（部分読み込みは許可しない）。 |
| **ファイルサイズ超過** | 10MB を超えるファイルは警告メッセージを表示して拒否する。 |

**エラーメッセージの例:**

```
// 不正な JSON
"JSONパースエラー: 行 15, 列 8 付近に不正な構文があります"

// 必須フィールド欠損
"必須フィールドが不足しています: heightCm, weightKg"

// 不正な値（CSV）
"行 4: width_cm の値が不正です（期待: 正の整数、実際: -50）"

// 不正な値（保存JSON）
"placements[2].cargoDefId: 参照先の荷物定義が見つかりません（id: uuid-9999）"

// ファイルサイズ超過
"ファイルサイズが上限（10MB）を超えています。ファイルを分割してください。"
```
