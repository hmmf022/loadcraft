# インタラクション設計

本ドキュメントでは、コンテナ積載シミュレータにおけるユーザー操作のフローと仕様を定義する。

3D 描画には WebGPU キャンバスとオービットカメラを使用し、荷物一覧からのドラッグ＆ドロップ、レイキャストによるオブジェクト選択、オイラー角による任意回転、重力制約（浮遊禁止）を基本要素とする。衝突（重なり）はシステム上許容し、最終的な判断はユーザーに委ねる。

---

## 1. ドラッグ＆ドロップフロー（新規配置）

CargoList から 3D キャンバスへ荷物を配置する一連のフローを以下に示す。

### ステップ 1: ドラッグ開始

ユーザーが CargoList 内の荷物アイテムをドラッグし始める。

- HTML5 Drag API の `dragstart` イベントで `dataTransfer` に `cargoDefId` をセットする。
- ドラッグプレビューとして荷物名と寸法（W x H x D）を表示する。

### ステップ 2: キャンバス領域への進入

ドラッグ中のカーソルがキャンバス領域に入る。

- `dragover` イベントでドロップを有効化する（`preventDefault()` を呼び出す）。
- 配置モードに切り替える。

### ステップ 3: キャンバス上でのマウス移動

カーソルがキャンバス上を移動するたびに以下の処理を行う。

- マウス位置からレイキャストを実行し、床面または既存荷物上のドロップポイントを特定する。
- グリッドスナップが有効な場合、最寄りのグリッド座標に吸着させる。
- 計算された位置にゴースト（半透明の荷物モデル）を表示する。
- ゴーストの色は有効な位置であれば緑、コンテナ範囲外であれば赤とする。
- 重力制約として、Y 軸方向で最も低い有効位置に自動的に落下させる。

### ステップ 4: マウスリリース（ドロップ）

カーソルを離した時点で配置を確定する。

- **有効位置の場合**: `PlaceCommand` を実行し、`VoxelGrid` に追加する。
- **無効位置の場合**: 配置をキャンセルし、短時間のエラーメッセージを表示する。
- いずれの場合もゴーストをクリアする。

### ステップ 5: アンドゥ

配置直後からアンドゥ（`Ctrl+Z`）が利用可能となる。

---

## 2. 既存荷物の選択フロー

キャンバス上に配置済みの荷物を選択する操作フローを以下に示す。

1. ユーザーがキャンバス上をクリックする。
2. クリック位置からレイキャストを実行する。
3. 配置済みの全荷物の AABB（軸平行バウンディングボックス）との交差判定を行う。
4. **ヒットした場合**: 最も手前（カメラに近い）荷物を選択し、アウトライン／グロウでハイライト表示する。
5. **ヒットしなかった場合**: 現在の選択を解除する。

### 選択中の荷物の表示

- 明るい色またはアウトラインによるビジュアルハイライトを適用する。
- PlacementControls パネルに選択荷物の情報（名称、位置、回転、寸法など）を反映する。

---

## 3. 既存荷物の移動フロー

選択済みの荷物をドラッグで移動する操作フローを以下に示す。

1. ユーザーが選択済みの荷物をクリックしたまま保持する。
2. 短い遅延（150ms）またはしきい値以上の移動量を検出した時点で移動モードに入る。
   - 荷物を `VoxelGrid` から一時的に除去する。
3. マウスをキャンバス上でドラッグする。
   - レイキャストにより新しい位置を特定する。
   - 新しい位置にゴーストを表示する。
   - ゴーストの色で有効性を示す（緑: 有効、赤: 警告/無効）。
4. マウスをリリースする。
   - **有効位置の場合**: `MoveCommand` を実行する（旧位置のボクセルをクリアし、新位置にフィルする）。
   - **無効位置の場合**: 荷物を元の位置に戻す。
5. アンドゥ操作により荷物を元の位置に復元できる。

---

## 4. 削除フロー

1. 3D ビューまたは PlacementControls で荷物を選択する。
2. `Delete` キーを押すか、PlacementControls 内の削除ボタンをクリックする。
3. `RemoveCommand` を実行する（`VoxelGrid` から該当荷物をクリアする）。
4. アンドゥ操作により削除した荷物を復元できる。

---

## 5. 回転操作フロー

### 5.1 サイドバーからの回転

1. 荷物を選択する。
2. PlacementControls に回転入力フィールド（rx, ry, rz: 度数法）が表示される。
3. ユーザーが回転値を変更する。
4. `RotateCommand` を実行する:
   - 旧ボクセルをクリアする。
   - 新しい回転角でボクセルを再計算する。
   - 新ボクセルをフィルする。
5. 3D ビューが新しい回転角で更新される。

### 5.2 キーボードショートカットによる回転

| キー | 操作 |
|------|------|
| `Q` / `E` | Y 軸回転 ±90° |
| `Shift+Q` / `Shift+E` | Y 軸回転 ±15° |
| `R` / `F` | X 軸回転 ±90° |

キーボードショートカットによる回転は、荷物が選択されている場合にのみ有効となる。

## 6. カメラ操作の詳細

### 6.1 マウス操作

| 操作 | マウス入力 |
|------|-----------|
| オービット（回転） | 左ボタンドラッグ |
| パン（平行移動） | 中ボタンドラッグ または 右ボタンドラッグ |
| ズーム | スクロールホイール |
| オブジェクト選択 | 左クリック（ドラッグなし） |

### 6.2 感度設定

| パラメータ | 値 |
|-----------|-----|
| オービット | 0.005 ラジアン / ピクセル |
| パン | 0.5 cm / ピクセル（ズームレベルに応じてスケーリング） |
| ズーム | スクロール 1 ステップあたり半径を 0.9 倍 / 1.1 倍 |
| スムースダンピング | オプションの慣性（lerp 係数 0.1） |

### 6.3 固定ビュー切替

- ビューボタンをクリックすると、プリセットの theta / phi / radius へスムースアニメーションで遷移する。
- アニメーション時間: 約 300ms。
- アニメーション中はユーザー入力を一時的に無効化する。

## 7. キーボードショートカット一覧

| キー | 操作 |
|------|------|
| `Delete` / `Backspace` | 選択中の荷物を削除 |
| `Ctrl+Z` | アンドゥ |
| `Ctrl+Shift+Z` / `Ctrl+Y` | リドゥ |
| `Q` / `E` | 選択荷物を Y 軸 ±90° 回転 |
| `Shift+Q` / `Shift+E` | 選択荷物を Y 軸 ±15° 回転 |
| `R` / `F` | 選択荷物を X 軸 ±90° 回転 |
| `Escape` | 選択解除 / ドラッグキャンセル |
| `1` - `6` | 固定ビュー切替（前面/背面/左側/右側/上面/アイソメトリック） |
| `G` | グリッド表示の切替 |
| `S` | グリッドスナップの切替 |

## 8. エラー状態の表示

| エラー状態 | 表示方法 |
|-----------|---------|
| **浮遊警告** | 荷物の支持率がしきい値を下回った場合、黄色の警告アイコンとメッセージを表示する。 |
| **範囲外** | 荷物がコンテナ境界の外に出ている場合、赤色のハイライトを適用する。 |
| **過積載** | 総重量が `maxPayload` を超過した場合、StatsPanel に赤色の警告を表示する。 |
| **ドラッグ中の無効位置** | ゴーストが赤色に変わり、カーソルが `not-allowed` に変化する。 |

すべての警告はノンブロッキングである。衝突はユーザー判断に委ねる設計のため、警告が表示されていてもユーザーは操作を続行できる。

---

## 9. ゴースト表示の仕様

ドラッグ中および移動中に表示されるゴースト（仮配置表示）の描画仕様を以下に定義する。

| 項目 | 仕様 |
|------|------|
| 透明度 | alpha = 0.4 による半透明レンダリング |
| 有効位置の色 | 緑色のティント |
| 無効位置 / 警告時の色 | 赤色のティント |
| レンダリング順 | メインの荷物レンダリング後に別パスで描画 |
| 深度バッファ書き込み | 無効（常に表示される） |
| 更新頻度 | ドラッグ中 60fps で更新 |
